棉花 Bot 是如何运转的？

棉花 Bot 并不是一个单一程序，而是由多个独立系统协同运行的结果。
你可以把它理解为“一个入口，多个功能模块”。

本页面会用通俗方式解释整个运行流程。

注：在不发生歧义的情况下，为了帮助普通用户理解，本文将框架一词改为系统一词。

Bot工作示意图​

A[QQ客户端] -->|发送消息| B(lltwobot)
B -->|转发| C[AstrBot]
B -->|转发| D[Koishi]
B -->|转发| E[Yunzai]
C -->|返回结果| B
D -->|返回结果| B
E -->|返回结果| B
B -->|发送回复| A

Bot运行在哪里​

棉花Bot运行在电脑上，此电脑安装了QQ客户端，用于登录QQ账号（这就是棉花Bot没有机器人标识的原因），接着使用其它工具通过运存抓取聊天记录到别的＂系统＂去，详情请看下文

QQ客户端：你看到的一切​

  • 你通过 QQ 发送消息
  • Bot 的回复也显示在 QQ
  • QQ 只是入口和显示界面

QQ 本身不处理任何功能逻辑。

lltwobot：消息中转桥梁​

lltwobot 负责：

  • 接收 QQ 发来的消息
  • 把消息同时转发给后端功能系统
  • 收集返回结果
  • 再发回 QQ

它不关心你发的是聊天、签到还是游戏查询。

它只负责“传递”。

三个功能系统：各自判断是否回应​

当一条消息被转发后：

系统负责什么
AstrBotAI 聊天、绘图等
Koishi常规指令、群功能
Yunzai游戏数据查询

它们都会同时收到消息。

每个系统都会判断：

  • 这条消息是不是我的指令？
  • 如果是 → 处理并返回
  • 如果不是 → 忽略

为什么有时候会出现多条回复？​

因为：

  • 每个系统独立判断
  • 它们之间不会互相协调

如果两个系统都认为“这是我的指令”，
就可能同时回复。

这属于正常现象。

温馨提示​

请你们使用bot时不要输入 @可爱的棉花 指令 ，而是 //指令（具体指令前缀请看指令菜单或者查看本文档后面部分） 或者 @可爱的棉花 你好 ，以防止多框架响应

总结​

棉花 Bot 的工作方式是：

  1. 你发送消息
  2. lltwobot 转发消息
  3. 不同系统判断是否处理
  4. 有结果就返回
  5. QQ 显示最终回复

它不是一个程序，而是一个组合系统。
